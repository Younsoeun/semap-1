# 페이지 
## 메인 페이지
- 고급지고 세련된 지구본으로 구성.
- 이 지구본은 줌인/줌아웃과 클릭 유지로 자연스럽게 돌릴 수 있고 마우스 커서를 특정 국가에 가져다 대면 그 부분이 밝게 빛나면서 인터렉션이 가능. 이 기능을 할 수 있는 실제 툴이 있으니 리서치를해서 가장 내 앱에 적합한 걸 만들어줘.
- 특정 국가를 클릭하면 해당 국가의 요약 페이지가 팝업으로 뜸.

## 국가 요약 팝업 구성
- 해당 국가에 대한 기본적인 설명과 관광지, 역사, 문화 등을 읽을 수 있음. 해당 내용은 상세 페이지 내용과 위키백과, 여러 뉴스를 통해 확인할 것.
- 국가 이름을 클릭하면 국가의 상세 페이지로 이동 가능

## 상세 페이지 구성
### 0. 필요한 페이지
- 메인 페이지
- 국가 요약 팝업
- 국가 상세 페이지
- 도시 상세 페이지
- 명소 카드 
- 국가별 문화 페이지
- 방문 루트 기록 페이지


## 국가 상세 페이지 구성
### 1. 나라 이름 및 지도
- 나라의 **실제 국경선 데이터 기반 지도**에 방문할 **명소**의 위치와 이름을 지도에 표시해줘
- 표시를 클릭하면 **명소 상세 페이지**로 이동하게 해줘

### 2. 도시 카드 배치 
- 도시 별로 이미지(위키피디아/위키미디어 커먼즈 사진) 1장을 사용
- 방문할 만한 명소 개수를 카드 좌측 하단에 표시 
- 도시 카드 클릭 시, 도시 상세 페이지로 이동 

## 도시 상세 페이지 구성
### 명소 카드 
- 도시에 위치한 방문할 만한 명소들을 한 눈에 보여줌
- 명소 별로 이미지 1장을 사용

### 명소 카드 내 구성
- 명소 별 **방문 일지 작성하기** 기능 보유
- 내가 해당 명소 방문 시의 생각, 추억을 **텍스트**와 **사진**으로도 기록할 수 있는 공간
-  로컬 데이터베이스를 쓸 수 있도록 하면 돼. 그리고 기본적으로 로컬 파일에 접근을 할거고. 이를 위해 아주 편리한 블로그 글 작성 인터페이스도 마련해줬으면 좋겠어. 그 외에 뭐 방문했던 도시, 먹은 음식 이런걸
- 마치 갤러리 형식으로 모아볼 수 있도록 해도 좋을 것 같아. 추가나 삭제는 자유롭게 할 수 있도록 마이크로 블로그 느낌을 살렸으면 좋겠어.
- 명소에 방문일지를 적으면 그걸 국가별로 모아서 볼 수도 있었으면 좋겠어.
- **방문 일지 작성하기**를 **완료**했다면 그 버튼은 **수정하기**로 바뀌고, 최초 1회 작성되었다면 전체 지구본에서 그 국가의 색이 바뀌어.(활성화되는 느낌) 이 활성화의 방식과 정도는 적절히 조정해
- 방문 일자도 기록할 수 있도록 해줘(방문 시작 일자와 종료일자 모두 기록하고 싶음)
- 명소에 대한 **설명**도 카드 내에 필수적으로 추가해
- 해당 명소의 **역사, 소개, 방문 팁** 등에 대해 자세히 다루도록해 
- 명소별 카테고리(미술/박물관, 운하/마을) 도 관리해. 유럽의 최대한 많은 나라에 공통적으로 사용할 카테고리이므로 최소한의 개수로 명소를 구분할 수 있도록해. 

## 방문 루트 기록 페이지
- 해당 페이지에서는 나라 별, 도시 별로 기록된 방문 시작일과 종료일을 한 눈에 볼 수 있도록해줘
- 방문 후보 국가에 대한 정보들을 모두 나열해놓고, 방문한 국가들을 방문 일자(오름차순)를 기준으로 고정되도록 

## 국가별 문화 페이지
- 내가 해당 국가를 방문할 때 주의해야 할 내용, 알아두면 좋은내용, 해당 국가를 100% 즐길 수 있도록 세세한 내용들도 설명해줬으면 해.
- 또한 해당 국가 출신의 사람과 친해질 때에도 참고할 내용이 있었으면 좋겠어.
- 인사, 언어 예절이나 유명한 음식이나 음식 문화, 식사예절, 사회적 매너, 공휴일이나 축제 같은 정보, 해당 국가의 인삿말이나 감사의 표현을 나타내는 말도 정리해줬으면 해
